# 安全说明

## 敏感信息

- 请勿将数据库密码、API Key 等写入 `config/json/` 或提交到版本库。
- 使用环境变量（如 `DB_PASSWORD`、各模型配置中的 `*_env` 字段）或本地覆盖配置管理敏感信息。
- `.env` 与含敏感信息的文件已列入 `.gitignore`，请勿强制添加。

## 报告安全问题

如发现与安全相关的漏洞或风险（如配置泄露、注入、权限问题等），请通过以下方式之一私下报告，避免在公开 Issue 中披露细节：

- 通过仓库维护者提供的联系方式（若已注明）直接联系。
- 或提交 Issue 并仅做概括性描述，注明「安全相关，将私下提供细节」，等待维护者回复。

我们会尽快评估并回复，修复方案确定后再考虑是否发布安全公告。
